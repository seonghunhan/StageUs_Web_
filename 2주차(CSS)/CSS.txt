HTML에서 스타일처리하면 CSS로 못덮음으로 HTML에서 스타일처리하면 안된다

디자인할때 한페이지에 폰트크기는 2가지로 가기

id는 1개만 사용해야함 (first를 여기 저기에 사용하면 안된다)
box는 여러개 가능

id가 클래스보다 우선순위이고, 그외에는 밑에코드가 우선순위

태그,id,class가 선택자라고 부름



이제 속성배워보자





디스플레이는 표현방법에 가깝다
(block -> 한줄자체를 그태그가 먹는것)<div>이나 <p>, <h1> 태그

(inline -> 전후 줄바꿈 없이 한 줄에 다른 엘리먼트들과 나란히 배치)<span>이나 <a>, <em> 태그

(inline block -> inline 엘리먼트에서 불가능하던 
width와 height 속성 지정 및 margin과 padding 속성의 상하 간격 지정이 가능해집니다.)
<button>이나 <input>, <select> 태그 





포지션은 원래위치상관없이 다른곳에 left,top등으로 보이게하려는것
(static -> 기본값이고 top,right,bottom,left,z-index속성들이 아무런 효과를 주지못함)
(relative -> 원래자리보다 다른곳에 띄우게 할때 사용)
(absolute -> 한곳에 때려박는것)
(fixed -> 스크롤움직여도 한곳에 계속 보이게 하는것)
