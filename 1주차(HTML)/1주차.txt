Web의 정의
World Wide Web을 줄여서 부르는말
인터넷 상에서 구동되는 여러 서비스의 종류 중 한 개
텍스트, 사운드, 사진, 동영상 등 멀티미디어 정보를 제공하는 시스템

Client(사용자)프로그램 -> 사용자가 사용하는프로그램(파이썬과정에서 주구장창 만든것)

웹을 배우면 서버라는 개념도 이해해야함


사용자 <-> 웹 브라우저 <-> 서버

사용자가 검색하면 웹브라우저에서 서버에서 보내준 CSS,자바스크립,HTML을 해석해줘서 사용자에게 넘김
(사용자가 아이피 URL창에 검색해서 서버로 넘기면 다시 웹브라우저로 코드를 넘겨준다)

근데 롤처럼 빠른속도를 요구하는데 이미지를 서버에서 가져오면 효율이 떨어지기때문에 그런것들은
사용자PC에 저장해놓는다

SEO (h,p,div)
Search Engine Optimaization
검색 엔진 최적화
웹페이지 수집하는 로봇이 이사이트가 괜찮은사이트인지 판단할 근거를 줘야하는데 태그를 기준으로함
그래서 로봇이 알아보기 쉽게 구조를 짜야한다

그래서 그냥 h1,p태그 이런걸로 폰트사이즈 키울수있는데 font이런걸로 키우고 h1,p태그는 SEO를 감안해서 사용한다
<!DOCTYPE html>
<html lang="kr">
<head>
</head>

<body>
    <div>							(로봇이 h1은 제목으로 인지)(h2는 부제목)(h3은 소제목)~h6 태그, 각각의 값들은 p태그로 넣는다
        <h1 style="font-weight:400; font-size:16px;"?제목입니다.</h1>   <!--SEO때문에 h1를 사용한다(우리사이트가 구글,네이버에 공개되기위하여)-->
        <p>내용들을 넣을예정입니다.</p>				(로봇이 내용으로 인지)
    </div>
</body>
</html>


로봇이 p1,h1등 안쓰면 제목,내용말고 한가지로 퉁쳐서 인지하므로 검색이 안될것이다

중요한건 h1부터 h6까지 쓰고 값들은 p에다가 쓴다



Semantic Tag
기본적으로 div와 동일하지만 태그이름이 다르다 (종류는 6가지)
<nav> (메뉴)
<main>(내용)
<section> (내용을 여러섹션으로 나눌때)
<article> (섹션보다 1개작은개념)
<header> (윗부분)
<footer> (맨밑부분)
왜 존재할까? -> div로해도되지만 맨위부분은 헤더, 맨밑부분은 푸터 이렇게 의미상 나누는것

아무튼 이렇게하는이유는 로봇이 구조를 잘 파악하여 간택당하길 바라는마음으로(그래야 검색엔진에 올리기 가능)

디스플레이는 표현방법에 가깝다
(block -> 한줄자체를 그태그가 먹는것)<div>이나 <p>, <h1> 태그

(inline -> 전후 줄바꿈 없이 한 줄에 다른 엘리먼트들과 나란히 배치)<span>이나 <a>, <em> 태그

(inline block -> inline 엘리먼트에서 불가능하던 
width와 height 속성 지정 및 margin과 padding 속성의 상하 간격 지정이 가능해집니다.)
<button>이나 <input>, <select> 태그 





포지션은 원래위치상관없이 다른곳에 left,top등으로 보이게하려는것
(static -> 기본값이고 top,right,bottom,left,z-index속성들이 아무런 효과를 주지못함)
(relative -> 원래자리보다 다른곳에 띄우게 할때 사용)
(absolute -> 한곳에 때려박는것)
(fixed -> 스크롤움직여도 한곳에 계속 보이게 하는것)
